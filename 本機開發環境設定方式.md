# 本機開發環境設定方式

## 目的
讓此專案在 Windows 本機快速啟動，並連接 Zeabur PostgreSQL，避免每次重來。

## 一次性前置需求
- 安裝 Node.js（`>= 20.9.0`）。
- 安裝 Docker Desktop（本專案可用本機 Docker PostgreSQL；若直接連 Zeabur 可不啟動本機 DB）。
- PowerShell 若被執行政策擋住 `npm`，優先使用 `npm.cmd`。

## 專案固定路徑
```powershell
Set-Location "c:\Users\55688\Desktop\cursor use\classinvit03"
```

## 套件安裝（Windows 安全做法）
```powershell
npm.cmd install
```

## 環境變數（.env）
檔案：`.env`

至少要有：
```env
DATABASE_URL=postgresql://<user>:<password>@<host>:<port>/<database>
NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

### 本機 Docker 資料庫（可選）
若使用專案內 `docker-compose.yml`，可用：
```env
DATABASE_URL=postgresql://classinvit:classinvit_local@localhost:5432/classinvit
NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

## 連接 Zeabur PostgreSQL（這次採用）
1. 到 Zeabur PostgreSQL 複製連線字串（本機通常使用 Public 連線）。
2. 貼到 `.env` 的 `DATABASE_URL`。
3. 驗證格式：
```powershell
npm.cmd run check:db-url
```
4. 套用 migration：
```powershell
npm.cmd run db:deploy
```
5. 匯入種子資料：
```powershell
npm.cmd run db:seed
```

> 已修正 `prisma/seed.ts` 的 `Category upsert` 衝突問題，`db:seed` 可重複執行。

## 啟動網站
```powershell
npm.cmd run dev
```

開啟：
- `http://localhost:3000`（優先）
- 若 3000 被占用，Next 會改用 3001，請看終端機顯示的 Local URL。

## 常見錯誤與快速處理

### 1) `npm : ... 無法載入 npm.ps1`
原因：PowerShell 執行政策擋 script。

解法 A（建議）：全部改用 `npm.cmd`。  
解法 B（可選）：
```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
```
之後重開 PowerShell。

### 2) `P1000 Authentication failed`
原因：`DATABASE_URL` 帳密或主機/埠不正確。  
解法：重新貼 Zeabur 連線字串，或確認本機 DB 帳密與 `.env` 一致。

### 3) `Another next dev server is already running`
原因：重複啟動 `next dev`。  
解法：直接用已在跑的 URL（通常 `http://localhost:3000`），或先停舊程序再啟動。

## 補充
- 若要看資料：`npm.cmd run db:studio`
- 若改回本機 Docker DB：
  - `npm.cmd run docker:up`
  - 若埠衝突，調整 `docker-compose.yml` 的對外埠，並同步修改 `.env` 的 `DATABASE_URL`。
