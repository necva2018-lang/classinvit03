# 專案階段報告（截至 2026-03-28）

本文件摘要目前程式庫已落地之功能、資料庫變更與部署注意事項，供內部對齊與版本留存。

---

## 一、階段目標與範圍

以 **Next.js 課程站台 + 後台管理** 為主體，完成：

- 課程 CTA 雙模式（購物車／補助課）之前後台一致行為。
- 課程內容與側欄資訊之 **後台可編、前台依內容顯示**。
- 列表篩選與類別與 **資料庫 Category** 同步。
- **課程「說明區段」**：後台可編排圖／影／文區塊，前台課程介紹分頁呈現。
- **首頁輪播**：可選背景影片（與圖片 poster 併用）；圖片欄位改為可空以支援純影片。
- **課程列表卡片**：每課可自訂無講師／無評價時副行文案；後台編輯入口收斂至「課程與單元 → 課程資訊」。
- **全站設定／頁尾**：多欄導覽連結（`顯示文字|網址`）、語系切換顯示開關等可於後台維護。
- Windows 本機開發體驗（終端機、環境檢查）之補強；`next.config` 補強 Turbopack root 與 Prisma 外部化設定以利穩定打包。

## 二、已完成項目（功能）

### 2.1 課程 CTA（`CourseCtaKind`）

- **購物車（CART）**：預設「加入購物車／立即購買」等文案與連結可於後台覆寫。
- **補助課（SUBSIDY）**：預設「預約諮詢／立即報名」語意；後台「價格、封面與本頁上架勾選」停用，儲存時不覆寫該等欄位（資料庫舊值保留）；前台不顯示定價／牌價／封面圖（改占位）、價格篩選與排序行為已區分補助課。
- 複製課程、匯出 MD、列表價格欄等已納入 CTA 類型之語意。

### 2.2 課程介紹分頁（前台）

- 「課程介紹／你可以學到／適合對象／學習前基本能力／學習前準備」：**後台未填則該區塊不顯示**；若整頁介紹區皆空，顯示簡短提示。
- 後台欄位與 Prisma 對應；並保留舊 Client 與 schema 不同步時之 raw 同步策略（learn／audience 相關）。

### 2.3 課程列表與搜尋

- 側邊／導覽類別與 **DB `Category`** 同步，連結使用 `?cat=<id>`（並保留舊有 tag 參數相容）。
- 補助課不納入「價格區間」篩選；「價格由低到高」排序時補助課排在後段。

### 2.4 前台側欄「課程資訊」

- 新增四則可選文字：**課程時長、單元結構、學習資源、完訓證明**。
- 後台於「基本資料」與「課程與單元 → 課程資訊」皆可編輯；**每欄有內容才顯示該列**；四欄皆空則不顯示整塊。
- **補助課**前台不顯示此整塊（文案仍存 DB，改回購物車課程後可生效）。

### 2.5 課程「說明區段」（圖／影／文）

- 資料存於 **`course_intro_blocks_json`**（JSONB）；後台於「課程與單元 → 說明區段」編輯。
- 前台課程介紹分頁依區塊類型渲染（圖片、內嵌影片、富文字等）；與既有「你可以學到」等純文字區塊並存。

### 2.6 首頁輪播（Banner）

- 欄位 **`banner_video_url`**：可填影片網址；**`imageUrl` 改為可空**，可作 poster 或與影片併用（至少圖或影片其一，由後台／種子約定）。
- 後台輪播管理與前台 `hero-swiper` 已對齊影片嵌入邏輯。

### 2.7 課程列表卡片文案

- 每課可選：**無講師時副行**、**尚無評價時說明**（對應 `course_listing_no_instructor_line`／`course_listing_no_reviews_line`）；留空則沿用站內預設字句。
- **僅於「課程與單元 → 課程資訊」** 與其他課程主檔欄位一併儲存（`saveCourseMeta`）；「編輯課程」基本資料表單不再寫入此二欄，頁面並附提示導向深度編輯。

### 2.8 全站設定、導覽與建置

- 頁尾多欄標題與連結、法務連結、語系切換顯示等由 **SiteSetting** 驅動；連結行格式為 **`顯示文字|網址`**（見 `lib/footer-nav.ts`）。
- **`next.config`**：`serverExternalPackages` 納入 Prisma；`turbopack.root` 鎖定專案根目錄；`images.remotePatterns` 擴充常見圖床／YouTube 縮圖以利封面與素材。

### 2.9 後台與工具

- 課程列表對補助課價格欄顯示「補助課（不適用）」。
- 後台儀表板捷徑、課程詳情／列表與 CTA 相關行為延續前一階段設計，並與本階段新欄位對齊。
- `dev.cmd`：便於在 Windows 以 cmd 啟動 dev（避開部分 PowerShell ExecutionPolicy 問題）。
- `.vscode/settings.json`：Windows 預設終端機改為 Command Prompt（可選共用）。
- 開發環境對 `DATABASE_URL` 之檢查已放寬（詳見 `lib/env-check.ts`）；`global-error` 文案已更新。

## 五、後續建議（非本階段必做）

- 補助課是否在後台 **一併隱藏**「側欄課程資訊」編輯區，可依產品決策再收斂。
- 種子資料（seed）可視需要為示範課程填入側欄四欄、說明區塊或列表卡片文案，以利新環境一眼看出效果。
